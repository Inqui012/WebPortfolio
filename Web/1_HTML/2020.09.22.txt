form 이어서
method속성에 get과 post 가 있다. get의 경우 선택한 데이터가 URL뒤에 붙어서 날아가고 post의 경우 데이터가 숨겨지고 날아가는 듯. 보안요소인가.
꾸준히 들어가는 name은 변수이름지정하는 개념. 실제 입력된 데이터가 아니라 'name에 입력된 데이터' 라는 형식으로 넘기기 위해 지정하는 느낌이다. a > b > 서버 같은. 서버에서 불러오는건 지정된 '네임에 들어있는 데이터'

input type="text"
속성으로 placeholder 이미지에서 사용했던것과 비슷함. 입력상자에 표시할 기초값을 입력. value와는 다르게 실제 데이터가 입력되어 있는것으로 인식하지 않는다. 반면 value로 값을 넣어놓으면 이것도 데이터로 인식함. 이 경우 유저가 데이터입력을 하면 value값 뒤로 이어진다.
required 필수속성. 해당 input의 입력이 필수적일때 사용하는 속성. 이게 있으면 submit 확인할때 데이터의 유무를 확인하나??
autofocus 페이지를 시작할때 자동으로 커서가 지정된다. 바로 입력할 수 있도록 하는 기능인듯.
readonly 읽기전용. 유저가 임의로 데이터를 변경할 수 없게 할 때 사용.

input type="email"
특별한 기능이 있는건 아니고. 모바일에서 다이얼패드를 열었을 때 자동으로 @가 나온다...? 이메일 입력 편하게 하려고 만들어진 인풋인듯. 모바일에서만 기능하고. 자동으로 영문+@ 키패드로 변경.

input type="url"
이메일과 동일, 영문+.com (길게입력으로 타 도메인 목록이 뜨는듯)

input type-"number"
숫자증감이 가능한 선택필드? 화살표두개로 숫자 조절이 가능. 음수 양수 넘나듬.
속성으로 min, max를 지정해서 숫자의 범위를 지정해줄 수 있다. 생년월일 같은거 넣을때 쓰나?
step 속성으로 버튼 클릭 한번으로 몇단위씩 넘어갈지 지정할 수 있다.
value로 기초값 설정도 가능.

input type="range"
미디어의 음량조절 바 같은애가 생긴다. min, max로 양쪽으로 움직일 수 있는 범위를 지정할 수 있고. step으로 똑같이 넘어갈 단위도 지정가능. value 로 초기값 설정도 가능. css 적용이 어렵기 때문에 자주 사용되지 않는다. 

input type="date"
일자를 선택할 수 있는 캘린더가 뜨네. 익스10 이상에서는 지원불가래, 수동입력만 가능하게 기능한다네.

input type="month", "week", "time"
년월일이 아니라 각각 월, 주, 시간을 지정할 수 있는 박스가 생긴다. 마찬가지로 IE10 이상에서는 안되고 있음.

input type="datetime-local"
연월일 시간까지 한번에 입력받을 수 있는 폼. 

input type="color"
색상선택을 할 수 있는 색 픽커 창이 뜬다. 포토샵에 있는 팔레트같은거. 얘도 ie10 이상이네.

<header> </header>
글로벌요소. 보통은 사이트의 상단부분. 웹페이지의 명칭이나 로고등이 여기에 들어간다. 머릿글? 명칭을 넣을때는 H태그 사용추천?

<nav> </nav>
navigation. 메뉴가 들어가는 곳. 목록이니까 보통은 ul, ol, li를 사용하는듯. 헤더에 들어가는 경우가 많긴한데 독립적으로도 사용할 수 있다.

<section> </section>
실제 컨텐츠가 들어가는 공간. 여러 섹션을 사용하기도 한다. 블록속성? 사용에 따라서 CSS의 작업이 편리해진다? 내용을 구분하기 위해 반드시 title들어가야한다. 보통 h2로 많이 사용하는듯. 헤더 아래에 위치하는 경우가 많이있어서. 권고사항이긴한데 필수적으로 하는게 좋다?

<article> </article>
독립적인 문서가 들어가는 태그. 섹션과 같이 제목을 넣어줘야한다. 섹션 아래에 있으니 섹션보다 한단계 아래 인 h3를 많이 사용하나? 홈페이지에 종속된 데이터가 아니라 다른곳에서 불러온 기사등을 게재하는 공간인듯. 섹션 안에 들어간다. 아티클 안에 섹션은 불가능.

<aside> </aside>
사이드메뉴. 글로벌요소로 사용할지는 선택적임.

<footer> </footer>
푸터. 보통은 회사 정보가 담겨있다. 글로벌요소. 여기의 정보가 명확해야한다.






* 원페이지 사이트. 하나의 페이지에서 모든 액션을 해내는것. 요즘 많이사용?
* 글로벌 요소. 전역특성. 하나의 웹사이트 안에서 공통으로 사용되는 부분을 일컫는 말인것 같은데. 페이지 안에서 헤더.html을 불러오고, 푸터.html을 불러오고, 실제로 내용이 바뀌는 body를 따로 표시하는 느낌. 
* 시맨틱 태그 블록속성. 안에들어가는 데이터의 양에 상관없이 한줄을 꽉 채운다.
* 인라인 속성 딱 자기만의 공간만 가지고 있는 태그.

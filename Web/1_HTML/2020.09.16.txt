태그. 셀프클로징 태그
src = 가져올 이미지의 경로. 자동완성 가능하다. ctrl 스페이스 누르면 해당 코드가 있는 폴더 안을 자동으로 보여준다.
alt = 지정된 이미지 로드가 안되었을 시에 보여주는것. 반드시 넣어주어야한다. 접근성때문에 웹 페이지를 읽어주는 기능일 경우 이 태그에 씌여있는 내용을 읽어서 이미지를 설명한다.

<video> </video>
<audio> </audio>
영상이나 음악의 주소는 태그 안에 src로 기입하고 태그 사이의 공간에는 해당 파일이 열리지 않을경우 표시할 문구를 써넣는다. 이미지태그의 alt와 같은 기능인가?
동영상의 경우 영상의 크기를 조정하는 width 와 height 태그가 있음. 이미지태그에도 동일하게 사용.
동영상이나 음악 재생을 위한 재생버튼등을 표시하려면 controls 태그가 필요하다.
autoplay 속성은 자동재생을 실행하는 명령...인데 현재는 기능하지 않는듯?
muted 음소거. autoplay와 함께 넣었을때 비디오의 경우에는 음소거로 영상이 자동재생되고 음성의 경우는 브라우저마다 자동재생 여부가 다르다. 크롬은 안되는듯.
자동재생은 현재는 불가능하다.
loop 반복. 영상이나 오디오가 종료된후 다시 재생하는 태그.
poster 이미지태그의 alt 같은 역할. 비디오를 불러오지 못할 때 대신 나타낼 이미지.





* 절대경로. 컴퓨터 입장에서 드라이브 에서 차례로 C > folder > folder 하는 형식.
프론트에서는 사용하지 않음. 백엔드에서 바꾸는듯.
* 상대경로. 작동하고 있는 html 파일을 기준으로 경로를 탐색. 동일 폴더인지, 상위, 하위인지를 찾아가게 만드는것. 상위폴더로 나갈경우는 ../ 를 사용. 하위폴더의 경우는 / 이후 그대로.

* via.placeholder
이미지를 넣을 장소인데 실제 이미지파일이 확정되지 않았을때 사용함.
/뒤의 숫자는 이미지의 가로세로 크기를 의미하는듯. 정방형만 되나?
직사각형도 된다. / 뒤에 이미지 크기를 000x000 식으로 지정하면 된다.

* contents delivery network
웹페이지 도메인의 경우 사이트의 용량이나 트래픽을 많이 신경써야하기 때문에 이럴경우 CDN이 많이 유용하세 사용된다. 요즘 많이 이쪽으로 사용하고 있는듯?
프론트에서는 이걸 많이 신경쓰는듯.

* 용량의 경우 이미이 > 폰트 

* 동영상의 경우 요즘에는 유투브에서 영상을 가져오는식으로 많이 한다. 공유에서 복사해오는 iframe 태그로 사용. 이것도 CDN형식.

* api (Apllication program interface)
제공받는 기능? 제공받는 기본 모듈같은 느낌. 모드 쓸 때 많이 쓰는 그런거.

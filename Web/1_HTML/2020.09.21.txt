
<form> </form>
action 속성이 바로 나오네. 이 폼태그로 얻은 데이터를 어디로 보낼지를 결정하는 속성인것 같아.

<fieldest> </fieldset>
폼에서 입력받은 데이터들을 그룹화하는 기능? 데이터상이 아니라 실제 웹에서도 박스로 구별이 되는것 같은데. 구별된다. 박스모양으로 구별해주고 레전드에 입력한 이름까지 표시하네

<legend> </legend>
그룹화된 필드셋의 이름을 지정하는 태그. 필수는 아니다.

<input>
폼에서 데이터를 입력받는 태그. 셀프클로징. 뒤에 /가 있는데 안적어줘도 되는듯. 입력할 수 있는 레터박스가 생긴다.
type 속성은 자동으로 생기고 입력받는 데이터의 종류를 정하는거인듯. text는 그대로 문자열. password 일 경우 자동으로 입력한 문자가 보이지 않도록 처리해준다.

radio는 동그라미 항목선택 중복 불가능, checkbox는 네모난 항목선택 중복선택 가능. 항목선택 속성의 경우 속성값 옆에 checked 를 넣어주면 처음부터 체크되어있는 상태의 항목을 지정할 수 있는것 같아.
radio 에서는 name 으로 그룹을 만들어줘야 중복선택이 안되게 한다. checkbox에서는 기능하지 않아. value 체크박스는 다중선택이기 때문에 유저가 선택한갑시 무엇인지 알아야 하기 때문에 버튼마다 값 이름을 지정해주는 속성인것같은. 
image 이미지 인풋. CDN 방식으로 웹상에서 이미지 URL을 가져와서 입력할 수도 있다.
button 버튼모양. 상호작용시 기능은 백엔드에서 구현하고 프론트에서는 모양만 잡아놓는 느낌. 버튼 안의 텍스트는 태그 안에서 value속성으로 넣어 줄 수 있다. 이경우 input 말고도 button 태그가 따로 있는데 잘 사용되지 않는다. 사용되는 경우가 다른듯.
file 파일첨부 기능. 클릭시 파일 찾는 윈도우 창이 실행된다. 이것도 후처리는 백엔드
submit 데이터를 확인하는 버튼. 다음 액션으로 넘어가는 기능을 구현할 떄 쓴다. 아무 기능이 없는 상태여도 누르면 화면을 새로고침 하는듯?
reset 기초값으로 입력되어 있는것이 아닌 유저가 입력한 내용들을 지우고 리셋하는 버튼. 쿠키로 인한 정보유출을 막기 위해 반드시 만들어주는게 좋다. 

title 마우스 오버했을때 나타나는 설명란에 나오는 문구. 
maxlength 그대로. 문자의 최대 길이. 지정된 숫자 이상으로는 입력되지 않음.
name 태그는 항목선택 이외에도 사용할 수 있음. 입력받는 데이터를 서버에서 관리하기 쉽도록 이름을 지정해주는 역할인가. 변수이름 지정하는 느낌. 프론트에서는 비워놓고 백엔드에서 자기 스타일에 맞게 수정해서 사용하는 듯. 서버쪽이니까...

<textarea> </textarea>
문자열을 입력할수 있는 공간이 있는 입력박스. 페이지 상에서 자유자재로 크기조절이 가능하다. 문장이 길어질경우 자동으로 스크롤바를 생성. 크기조절은 cols, rows 로 가능한데 현재는 이것보다는 css에서 따로 지정해주는방식을 쓴다.
태그 사이에 텍스트박스에 자동으로 표시될 문구를 써놓을 수도 있다. 코드상의 탭도 인식하네. 이건 한줄로 써야 좋을듯. <pre>와 같은 느낌이다.

<label> </label>
input 옆에 텍스트 표시할때 쓰는건가? for 속성에 값을 넣어서 각각 대응되는 input id 와 대응시키도록 만들 수 있다. 대응시키면 선택박스가 아니라 텍스트를 눌러도 자동으로 대응되는 input이 선택된다. 박스랑 라벨이랑 순서가 바뀌어 있어도 자동으로 대응되는 애로 선택하네.

<select> </select>
드롭다운 형식의 선택문을 만들어낸다. 선택문을 입력할때는 태그 안에 option 태그를 넣는다.

<option> </option>
셀렉 태그의 목록을 지정하는 태그. 선택되어 있지 않은 초기값을 표시하는 맨 윗 옵션의 value는 비워놓는게 일반적인듯. 이 경우 선택되지 않는다. value 속성은 프론트에서는 비워놓고 백엔드에서 사용에 따라 수정한다.

<optgroup> </optgroup>
input 의 fieldest 처럼 그룹화 해주는 태그. 목록상에 공백을 넣어서 그룹화를 알아보기 쉽게 해주는듯. 
label 속성을 줘서 공백이 아니라 그룹화의 이름을 표시하도록 할 수도 있다.
selected 값을 줘서 기초값을 설정해줄 수 있다. checked와 동일.


* 테이블 태그 보더값을 테이블 태그안이 아니라 스타일 태그 안에서 줄 수도 있네.

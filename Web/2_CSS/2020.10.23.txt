
* 선택자, 그룹핑 방식이라고 한다. 컨텐츠를 그룹으로 묶어서 CSS를 주는 형식이 그룹핑. 그룹이 다르면 같은 속성을 여러번 쓰게 되겠네. 보기에는 편할지도 모르지만.
특정 속성의 CSS를 미리 만들어놓고 적용하고 싶은 태그에 클래스를 부여하는게 모듈형. 말 그대로 모듈. 세분화를 잘해야 쓰기 편할듯. 너무 세분화되면 클래스 하나하나 찾아가느라 힘들것 같은 이미지.
    태그:nth-child()     자식들중에서 () 안에 지목된 태그를 잡아낸다.  
    태그:nth-of-type()   타입중에서 () 안에 지목된 태그를 잡아낸다. 타입이니까 동일 태그들중에서 괄호의 번호에 적용되는 애들을 전부 잡아내는듯.

    ex> div h1:nth-child(2n) > div 안에 있는 h1 중에서 2n번째만
    ex> div h1:nth-of-type(3) > div 안에 있는 h1 중에서 세번째만

* 태그는 그냥 [태그], 클래스는 .[클래스명], 아이디는 #[아이디명]
* div1 ~ div2 > div1 태그와 동일선상에있고 div1보다 뒤에 오는 모든 div2 태그
* div1 + div2 > div1 태그와 동일선상에있고 div1 바로 뒤에 따라오는 모든 div2 태그 

* 태그[type=''] 으로도 지목할 수 있음. 같은 태그들중에서 특정한 타입을 가진 태그를 지정할때 사용할듯.
    ex> input 태그들중에서 패스워드 인풋만을 지목하거나.

* {cursor: pointer / default / crosshair / ... ;}
    커서 모양 바꾸는 속성.

* 태그명:checked 
    라디오 박스나 체크박스의 경우 체크되었을 때 작동하는 CSS를 줄 수 있다.

* { -ms- / -webkit- / -moz- / -o- / transition-duration: 시간;}
    변화가 시간값 만큼 지속된다. 간단하게 애니메이션을 넣을 수 있는듯. 기본수치 -> 지정된 수치로 이동 하는 느낌. 호버나 체크랑 같이 쓸일이 많은가? 아니다 단독으로도 쓰긴하겠다. 
    앞의 접두사는 브라우저에 따른 설정값들인듯. 마소, 크롬 , 모질라, 오페라. 베타버전이라고 생각하면 될듯. 차후에 변경될수도 있고 안될수도 있고.

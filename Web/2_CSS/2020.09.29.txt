
* {display: (none/inline/block/inline-block);}
    display: none 요소를 보여주지 않는다.
    display: block 블록속성. 가로영역을 모두 채운다.
    display: inline 인라인속성. widht, height 속성들을 가질 수 없다.
    display: inline-block 인라인 요소와는 달리 '블록 요소'처럼 widht 와 height 속성 지정 및 margin 과 padding 속성의 상하간격 지정이 가능하다. 

* {margin-top/bottom/left/right : ();}
    그냥 margin 으로 넣으면 상화좌우 한번에 다 넣을 수 있다.(축약법) 값을 두개 넣어주면 앞의 값이 위, 아래의 마진, 뒤의 값이 왼쪽 오른쪽의 마진. 값을 넣는 순서대로, [위(아래) 오른쪽(왼쪽) 아래 왼쪽] 순으로 들어감.
* margin auto
    auto 는 남은 공간을 균등하게 알아서 배분하라는 속성. 중간에 배치할때 쓸때 편할듯?
    ex) margin 0 auto = 상하로 0, 양옆의 마진을 균등하게 가진다는 태그.
    이건 display 가 block 일때만 작용한다.
    ? 블록이 아니어도 된다?

* 중앙정렬 할때 블록속성이면 마진 0 auto 를 쓰면 되고, 인라인이나 인라인 블록일경우에는 text-align 속성을 사용할 수 있다.

* 축약법은 padding 에도 똑같이 적용됨.

* .html 에서 .css 로 옮겨가면서 작업할때 클래스나 아이디는 오타방지를 위해 복붙하는게 좋음.

* 두 영역의 margin 은 상쇄된다. 두 마진이 20px, 30px 이라고해서 영역사이의 거리가 50px이 되지는 않음. 큰 숫자인 30px이 남아있는다. 때문에 실무에서 margin을 사용할때는 한쪽 방향으로만 값을 주는일이 좋다고 하네. 양 옆은 별로 상관 없는듯.

* 두 영역의 margin 과 padding 은 가산되서 적용됨. 마진 10, 패딩 10 일때 영역사이의 공간은 총 20이 됨.

* 중앙은 0 auto. 오른쪽으로 밀면 왼쪽 공간에 auto. 반대면 왼쪽으로 밀고.

* 태그의 width 값에 padding 값은 들어가는데 border 값은 안들어가나? 아 들어가네

* border 속성은 선 크기랑 선 종류가 반드시 들어가야 적용된다. 점선, 실선 등등...

* CSS 에서 크기지정 해줄때 box-sizing 속성을 넣어주면 패딩값과 보더값을 따로 계산해서 맞춰주지 않아도 지정된 width 값 안에서 패딩과 보더를 설정한다? width 값을 알아서 지정해주는듯.

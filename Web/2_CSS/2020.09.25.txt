
* CSS를 먼저 만들어놓고 여러개의 태그에 중복적용하는걸 모듈형식이라고 함. 이렇게되면 코드 작업이 빨라지는 장점이 있는데 문제가 생겼을경우 모든 페이지가 망가질 수도 있음. 유지보수가 힘들어짐. 모듈형식은 시안이 완벽할때 사용하는듯? 

* CSS에서 클래스 지정할때 [.클래스1.클래스2] 형식으로 써주면 두 클래스를 모두 가지고 있는 것만 인식하게 된다.

* 클래스와 아이디에 쓸 수 있는 이름에는 규칙이 있다.
    1. 첫 글자는 알파벳으로 시작해야 한다. (a~z 또는 A~Z)
    2. 두 번째 부터는 알파벳과 숫자 그리고'-' 또는 '_'가 사용될 수 있다.
    3. 대소문자를 구분한다.
    PascalCase (파스칼 케이스) / 연결된 단어의 경우 첫글자를 대문자로. 쌍봉낙타 표기법.
    camelCase (카멜 케이스) / 첫단어는 무조건 소문자, 뒤에 붙는 단어의 첫글자를 대문자로. 단봉낙타 표기법.
    snake_case (스네이크 케이스) / 단어구분을 _ 로 한다.
    kebab-case (케밥 케이스) / 단어구분을 - 로 한다.

* 일부태그는 스스로의 CSS 기본값이 없을경우 부모태그의 CSS를 상속받아 적용된다. 실무를 할때는 모든 CSS 를 초기화하고 시작하기 때문에 이런경우는 거의 없다.

* CSS 지정할때 * 로 지정하면 모든 태그를 지정하도록 한다.

* HEX 코드 = 16진수 코드

* CSS 코드는 세미콜론 구분하면서 한줄로 쓴다. 

* 태그를 동시에 잡을 때는 [태그1 태그2] 형식을 사용. 부모태그1 아래의 태그2 를 인식한다. 주의할점은 아래에 있는 모든 해당태그를 인식하기 떄문에 바로 아래 하나만 인식하는게 아니다.

* 여러 다른 태그에 동일한 CSS를 적용할때는 [태그1 , 태그2] 형식을 사용. 콤마로 구분한다.

* 여러 같은 클래스중에 특정 태그에 적용되어 있는 클래스를 지정하고 싶을떄는 [태크.클래스명] 으로 지정할 수 있다.

* 자식태그를 지정할때는 [태그1 > 태그2]를 사용. 부모태그 바로 아래에 있는 태그2 만을 지정해서 인식한다. 띄어쓰기랑 다르니까 주의. 자식위치에 있는 해당 태그를 전부 인식하네.

* [태그1(클래스1) + 태그2] 는 태그1 또는 클래스1 의 바로 다음에 나오는 태그2 를 지정.
* [태크1(클래스1) ~ 태그2] 는 태그1 또는 클래스1 과 같은 선상에 있는 태그들을 지정.

EX]
태그1 태그2 > 태그3 , 태그4 , 태그.클래스1
이경우는 [태그1 아래의 태그2 바로 아래의 태그3]과 태그4, 태그중에서 클래스1을 가진 태그에 동일 CSS를 지정할 수 있음.
